# Contributing to Kynto Intel

First of all, thank you for your interest! By contributing to Kynto Intel, you are helping build a more powerful and accessible database management tool.

You can find the official repository here: [https://github.com/Kynto-intel/kynto-local-db](https://github.com/Kynto-intel/kynto-local-db)

## 1. How You Can Help

### Reporting Bugs
If you encounter any issues (e.g., path errors on Windows or app crashes), please open a **GitHub Issue**. Provide a brief description of:
* What you were doing.
* What happened (error messages or screenshots).
* What you expected to happen.

### Feature Requests
Do you have an idea for a new database connection, UI improvement, or AI integration? Open an issue and tag it as an "enhancement."

### Submitting Code (Pull Requests)
1. Fork the repository.
2. Create a new branch for your changes (`git checkout -b feature/your-feature-name`).
3. Commit your changes.
4. Create a Pull Request (PR) explaining your changes.

## 2. Local Development Setup

To run Kynto Intel locally and start developing:

1. Clone the repository:  
   `git clone https://github.com/Kynto-intel/kynto-local-db.git`
2. Install dependencies:  
   `npm install`
3. Start the app in development mode:  
   `npm start`
4. Build the production executable:  
   `npm run dist`

## 3. Code of Conduct
Please be professional, respectful, and constructive. We are building this community to help developers work more efficiently.
