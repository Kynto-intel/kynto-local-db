Verwendung in Projekten

Wichtig: Jeder API Key wird nach Verwendung mit [VERWENDET] gekennzeichnet. Du kannst ihn dann manuell löschen wenn er nicht mehr nötig ist.

1. Als Bearer Token im Header:

Authorization: Bearer [dein_api_key]

2. JavaScript Fetch Beispiel:

const response = await fetch('http://localhost:54321/data/users', {
method: 'GET',
headers: {
'Authorization': 'Bearer ABC123DEF...'
}
});

3. cURL Beispiel:

curl -H "Authorization: Bearer ABC123DEF..." \
  http://localhost:54321/data/users

⚠️ Sicherheit: Teile deinen API Key niemals öffentlich! Der Key funktioniert wie ein Passwort. Wenn die Verwendung verdächtig aussieht, lösche den Key und generiere einen neuen.

API Keys Dokumentation

Was ist ein API Key?

Ein API Key ist ein eindeutiger Token, den du generierst, um von externen Anwendungen auf deine Kynto-Instanz zuzugreifen. Er funktioniert wie ein Passwort und sollte genauso sicher behandelt werden.

Was kann ich mit einem API Key tun?

GET /data/:table

Daten aus einer Tabelle abrufen

POST /data/:table

Neue Datensätze in eine Tabelle einfügen

PUT /data/:table/:id

Einen Datensatz aktualisieren

DELETE /data/:table/:id

Einen Datensatz löschen

POST /query

Benutzerdefinierte SQL Queries ausführen (SELECT, INSERT, UPDATE, DELETE)

Wie verwende ich den API Key?

Header:

Authorization: Bearer YOUR_API_KEY

ℹ️ Hinweis: Ersetze YOUR_API_KEY mit deinem generierten Key. Die Standard-URL ist http://127.0.0.1:54321, aber passe Host und Port an falls nötig.

Vollständiges Fetch-Beispiel (GET):

const API_URL = 'http://127.0.0.1:54321'; const API_KEY = 'YOUR_API_KEY'; const response = await fetch(`${API_URL}/data/users`, { method: 'GET', headers: { 'Authorization': `Bearer ${API_KEY}` } }); const data = await response.json(); console.log(data);

Fetch-Beispiel (Daten einfügen POST):

const API_URL = 'http://127.0.0.1:54321'; const API_KEY = 'YOUR_API_KEY'; const response = await fetch(`${API_URL}/data/users`, { method: 'POST', headers: { 'Authorization': `Bearer ${API_KEY}`, 'Content-Type': 'application/json' }, body: JSON.stringify({ name: 'Max Mustermann', email: 'max@example.com' }) }); const data = await response.json(); console.log(data);

Sicherheit

⚠️ Wichtig: Niemals den API Key in öffentlichen Repositories (GitHub, GitLab, etc.) hochladen! Nutzer mit deinem Key können auf alle deine Daten zugreifen.

Best Practices:

  • Speichere den Key in Umgebungsvariablen (.env)
  • Verwende ihn nicht in Frontend-Code
  • Rotiere Keys regelmäßig (lösche alte, generiere neue)
  • Überwache die "Verwendet"-Spalte auf verdächtige Aktivitäten

Troubleshooting

401 Authentifizierung fehlgeschlagen

Der API Key ist ungültig oder wurde gelöscht. Generiere einen neuen Key.

403 Zugriff verweigert

Du versuchst eine nicht erlaubte Operation (z.B. DROP TABLE). Nur SELECT, INSERT, UPDATE, DELETE und CALL sind erlaubt.

500 Server-Fehler

Es gibt ein Problem mit dem Request. Überprüfe die Tabellennamen und Spaltennamen auf Tippfehler.
